Ajuste de credencial inválida Iconatus

CRMs Homologados pela iConatus

A credencial é a "chave de acesso" (como senhas ou tokens) que permite a conexão segura entre o CRM da imobiliária e o Marketplace. Ela garante que o integrador consiga coletar e transferir os dados de imóveis e contatos de forma autenticada.

Integração via XML


Integração via XML realizar o preenchimento através desse link do parceiro integrador:


http://adm.iconatus.com.br:7070/account/login


Na primeira etapa, insira o usuário e a senha fornecidos pela Iconatus no início da integração. Caso não os possua, acione o suporte 5A.

A próxima etapa consiste em preencher os dados de acesso ao CRM.

Ao finalizar o cadastro, a mensagem "Concluído com sucesso" será exibida. No entanto, ela confirma apenas o registro, sendo necessária a validação da Iconatus para garantir o ajuste das credenciais.

Integração via API


Importante:

Integração via API o ajuste precisa ser realizado através de abertura de chamado com o CRM da imobiliária.

Modelo de chamado para o CRM:


“Utilizamos hoje junto à Iconatus as credenciais abaixo para compartilhar dados de proprietário e imóvel. No entanto, a chave API está inválida, impedindo o retorno dos dados. Por favor, poderiam corrigir a chave atual ou fornecer uma nova?”


Importante: Enviar o link da chave API compartilhada com a Iconatus no início da integração.

Preenchimento Usuário

http://adm.iconatus.com.br:7070/account/login


Na primeira etapa, insira o usuário e a senha fornecidos pela Iconatus no início da integração. Caso não os possua, acione o suporte 5A.
